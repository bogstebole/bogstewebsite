## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Canvas pixel unit — `CHARACTER.PIXEL_SIZE = 4` (source: lib/constants.ts) |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Canvas pixel grid uses `PX = 4` as its atomic unit. All sprite geometry is in multiples of 4px: canvas buffer `512 × 384` (`COLS 128 × PX 4`, `ROWS 96 × PX 4`). CSS display size is independent of the buffer size.
- The overlay covers the full viewport: `position: absolute; inset: 0` — no spacing tokens apply to the overlay container itself.
- The close-up sprite display size uses `height: 80vh; width: auto` for Phase 1 placeholder. Phase 2 overrides this during the lunge animation.

## Interaction Contract

Phase 1 introduces no user interactions beyond the existing canvas. The close-up sprite overlay is passive in this phase — it renders and waits. Phase 2 adds the lunge trigger.

### Overlay Behavior

| State | Behavior |
|-------|----------|
| First visit (`intro_seen` absent in localStorage) | Overlay mounts: `CloseupSprite` renders absolutely over the canvas |
| Repeat visit (`intro_seen === "true"`) | Overlay never mounts — canvas is immediately interactive |
| localStorage unavailable (incognito / blocked) | Treat as first visit — overlay mounts every time |

### Overlay Visual Contract

| Property | Value | Rationale |
|----------|-------|-----------|
| `position` | `absolute` | Sits over the canvas, same pattern as `IntroSequence` |
| `inset` | `0` | Full viewport coverage |
| `z-index` | `30` | Above canvas elements (matches existing overlay stack) |
| `background` | `transparent` | No backdrop for Phase 1 — sprite floats over `#DAD9D2` canvas background |
| `display` | `flex` | Centers the sprite |
| `alignItems` | `center` | Vertical center |
| `justifyContent` | `center` | Horizontal center |
| `pointerEvents` | `auto` | Overlay captures input (click to dismiss — temporary Phase 1 behavior) |

### Close-up Sprite Rendering Contract

| Property | Value | Source |
|----------|-------|--------|
| Canvas buffer width | `512px` (`COLS 128 × PX 4`) | CONTEXT.md locked decision |
| Canvas buffer height | `384px` (`ROWS 96 × PX 4`) | CONTEXT.md locked decision |
| CSS display height | `80vh` | RESEARCH.md Phase 1 placeholder recommendation |
| CSS display width | `auto` | Maintains aspect ratio — Phase 2 overrides |
| `imageRendering` | `pixelated` | Preserves crisp pixels at any display scale (source: locked decision) |
| Pixel data format | 96-row string array, each row exactly 128 characters | CONTEXT.md / RESEARCH.md pattern |
| Color key lookup | `PALETTE` imported from `PixelCharacter.tsx` | Must not be redefined |
| Drawing method | `ctx.fillRect(col * PX, row * PX, PX, PX)` per non-transparent pixel | RESEARCH.md Pattern 2 |
| `CLOSEUP_FRAME` declaration | Module-level `const` — outside component function body | RESEARCH.md anti-pattern note |

### Expression & Pose Contract (art direction)

| Element | Specification |
|---------|--------------|
| Grid | 96 rows × 128 columns |
| Crop | Face + shoulders only — not full body |
| Orientation | Forward-facing — no three-quarter turn |
| Eyes | Heavy-lidded, squinting — intense focused stare |
| Mouth | Slight smirk at corner — confident, self-aware energy |
| Cap | Same cap as walking character — fully visible, straight-on, brim not casting shadow |
| Tone | Not comedic, not purely menacing — confident and self-aware. The intensity is the hook; the smirk is the personality. |

## Gate Logic Contract

The localStorage gate is a behavior contract, not a visual one. Included here so the executor has a single source of truth.

| Decision | Value | Source |
|----------|-------|--------|
| Storage key | `intro_seen` | CONTEXT.md locked decision |
| Flag value | `"true"` (string) | CONTEXT.md locked decision |
| When to set | In `onDismiss` callback — NOT at mount | CONTEXT.md locked decision |
| localStorage unavailable | Treat as first visit; fail silently | CONTEXT.md locked decision |
| Dev reset | Clear `intro_seen` from DevTools manually | CONTEXT.md locked decision — no URL bypass param |
| Version keying | None — flag never resets once set | CONTEXT.md locked decision |
| Existing key to migrate | `boule-visited` → `intro_seen` | RESEARCH.md Pattern 4 |
